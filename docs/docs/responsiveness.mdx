# Responsiveness

## Table of contents

As mentioned in the [quickstart section](/quick-start), the Table component is able to scale to various page sizes (see [breakpoints](/quick-start#breakpoints)) to ensure a good experience for all users.

You are able to:

-   [**Hide some columns**](#hide-columns) below a specific breakpoint
-   Show a [**mobile version**](#mobile-version) of the table designed for small devices.

## Mobile version

The mobile version of the table is enabled by default under `BREAKPOINT.MD`. Since every table in the docs is renderer using **reTables**, you can see this by yourself heading over to one of the `Live examples`.
In the following example, for demonstration purposes, we will force the mobile view on all screens.

To customize this behavior use the `breakpoint` prop of `<Table />` component.
